# Справочник команд — QA Automation Sandbox

## Запуск тестов

### По набору
```bash
# Smoke (~30 сек)
npx playwright test e2e/smoke.spec.ts --project=chromium

# Sanity (~15 сек)
npx playwright test e2e/sanity.spec.ts --project=chromium

# Chromium only (~5 мин)
npx playwright test --project=chromium

# Full suite (~30 мин)
npm test

# Component tests (~5 сек)
npm run test:components

# PBT tests
npm run test:pbt

# DB tests
npm run test:db

# Gherkin tests
npm run test:gherkin

# Visual regression tests
npx playwright test e2e/visual.spec.ts --project=chromium

# Update visual snapshots
npx playwright test e2e/visual.spec.ts --project=chromium --update-snapshots
```

## Виды нотаций тест-кейсов

### 1. Gherkin (Cucumber)

```gherkin
# features/auth.feature
Feature: User Authentication

  Scenario: Login with valid credentials
    Given I am on the login page
    When I enter "alice@buzzhive.com" and "alice123"
    And I click login button
    Then I should be logged in

# Запуск
npm run test:gherkin
```

### 2. JSON (Table-Driven)

```json
// test_data/login_cases.json
[
  {"email": "alice@test.com", "password": "alice123", "expected": 200},
  {"email": "invalid@test.com", "password": "wrong", "expected": 401},
  {"email": "", "password": "alice123", "expected": 422}
]
```

```python
# python/tests/test_login.py
import json
import pytest
import requests

BASE_URL = "http://localhost:8000/api"

with open("test_data/login_cases.json") as f:
    cases = json.load(f)

@pytest.mark.parametrize("case", cases)
def test_login(case):
    response = requests.post(
        f"{BASE_URL}/auth/login",
        json={"email": case["email"], "password": case["password"]}
    )
    assert response.status_code == case["expected"]

# Запуск
npm run test:python  # или pytest python/tests/test_login.py -v
```

### 3. YAML (Table-Driven)

```yaml
# test_data/auth.yaml
- name: "Valid login"
  method: "POST"
  path: "/auth/login"
  body:
    email: "alice@test.com"
    password: "alice123"
  expected_status: 200

- name: "Invalid password"
  method: "POST"
  path: "/auth/login"
  body:
    email: "alice@test.com"
    password: "wrong"
  expected_status: 401
```

```python
import yaml
import pytest
import requests

cases = yaml.safe_load_all(open("test_data/auth.yaml"))

@pytest.mark.parametrize("case", cases)
def test_auth(case):
    response = requests.request(
        case["method"],
        BASE_URL + case["path"],
        json=case.get("body")
    )
    assert response.status_code == case["expected_status"]
```

### 4. Python List (Inline Table-Driven)

```python
# test_login.py
import pytest
import requests

BASE_URL = "http://localhost:8000/api"

# (email, password, expected_status)
TEST_CASES = [
    ("alice@test.com", "alice123", 200),
    ("invalid@test.com", "wrong", 401),
    ("", "alice123", 422),
]

@pytest.mark.parametrize("email,password,expected", TEST_CASES)
def test_login(email, password, expected):
    response = requests.post(
        f"{BASE_URL}/auth/login",
        json={"email": email, "password": password}
    )
    assert response.status_code == expected
```

### 5. CSV (Table-Driven)

```csv
# test_data/login.csv
email,password,expected_status
alice@test.com,alice123,200
invalid@test.com,wrong,401
,alice123,422
```

```python
import csv
import pytest
import requests

BASE_URL = "http://localhost:8000/api"

@pytest.mark.parametrize("row", csv.DictReader(open("test_data/login.csv")))
def test_login_csv(row):
    response = requests.post(
        f"{BASE_URL}/auth/login",
        json={"email": row["email"], "password": row["password"]}
    )
    assert response.status_code == int(row["expected_status"])
```

## Сравнение нотаций

| Нотация | Читаемость | IDE Support | Для больших данных | Пример использования |
|---------|-------------|--------------|-------------------|----------------------|
| **Gherkin** | ✅ Отлично | ✅ | ❌ Сложно | BDD, поведение |
| **JSON** | ❌ Плохо | ✅ | ✅ Отлично | API ответы, CI |
| **YAML** | ✅ Хорошо | ✅ | ✅ Отлично | Конфиги, тесты |
| **Python list** | ✅ Хорошо | ✅ | ⚠️ Средне | Быстрые тесты |
| **CSV** | ✅ Хорошо | ✅ | ✅ Отлично | Excel → тесты |
| **YAML Suite** | ✅ Отлично | ✅ | ✅ | Полные suites |

## Когда что использовать

| Ситуация | Рекомендуемая нотация |
|----------|----------------------|
| BDD с бизнесом | Gherkin (Cucumber) |
| Простые API тесты | Python list |
| Много тестов (50+) | YAML или JSON |
| Excel → тесты | CSV → Python |
| Документирование | YAML Test Suite |
| CI/CD pipeline | JSON |
