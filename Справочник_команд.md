### 2. Python (Pytest + Table-Driven) — БЕЗ Gherkin/Cucumber

**Установка:**
```bash
pip install -r python/requirements.txt
# pytest, pytest-yaml, requests
```

**Реализация тех же тестов:**

```python
# python/tests/test_auth.py
import pytest
import requests

BASE_URL = "http://localhost:8000/api"

# Вариант 1: Python List
TEST_CASES = [
    # (email, password, expected_status, expected_result)
    ("alice@buzzhive.com", "alice123", 200, "logged_in"),
    ("alice@buzzhive.com", "wrongpassword", 401, "error"),
]

@pytest.mark.parametrize("email,password,expected_status,expected_result", TEST_CASES)
def test_login(email, password, expected_status, expected_result):
    response = requests.post(
        f"{BASE_URL}/auth/login",
        json={"email": email, "password": password}
    )
    assert response.status_code == expected_status
    
    if expected_result == "logged_in":
        assert "access_token" in response.json()
    elif expected_result == "error":
        assert "error" in response.json() or response.status_code == 401

# Вариант 2: JSON
# python/tests/test_data/login_cases.json
"""
[
  {"email": "alice@buzzhive.com", "password": "alice123", "expected": 200},
  {"email": "alice@buzzhive.com", "password": "wrongpassword", "expected": 401}
]
"""
import json
with open("test_data/login_cases.json") as f:
    cases = json.load(f)

@pytest.mark.parametrize("case", cases)
def test_login_json(case):
    response = requests.post(
        f"{BASE_URL}/auth/login",
        json={"email": case["email"], "password": case["password"]}
    )
    assert response.status_code == case["expected"]

# Запуск
pytest python/tests/test_auth.py -v
# или через npm
npm run test:python
```

**Инструменты:** `pytest` + `requests` + `pytest-yaml` (опционально)

### 3. YAML (Data-Driven) — БЕЗ Gherkin/Cucumber

**Те же тесты в YAML:**

```yaml
# python/tests/test_data/auth.yaml
- name: "Login valid credentials"
  description: "User can login with correct email/password"
  method: "POST"
  path: "/auth/login"
  body:
    email: "alice@buzzhive.com"
    password: "alice123"
  expected_status: 200
  expected_field: "access_token"

- name: "Login wrong password"
  description: "User cannot login with wrong password"
  method: "POST"
  path: "/auth/login"
  body:
    email: "alice@buzzhive.com"
    password: "wrongpassword"
  expected_status: 401

- name: "Login missing email"
  description: "Validation error on empty email"
  method: "POST"
  path: "/auth/login"
  body:
    email: ""
    password: "alice123"
  expected_status: 422
```

```python
# python/tests/test_yaml_runner.py
import yaml
import pytest
import requests

BASE_URL = "http://localhost:8000/api"

with open("test_data/auth.yaml") as f:
    cases = list(yaml.safe_load_all(f))

@pytest.mark.parametrize("case", cases)
def test_auth_yaml(case):
    response = requests.request(
        case["method"],
        f"{BASE_URL}{case['path']}",
        json=case.get("body")
    )
    assert response.status_code == case["expected_status"]
    
    if case.get("expected_field"):
        assert case["expected_field"] in response.json()

# Запуск
pytest python/tests/test_yaml_runner.py -v
```

**Инструменты:** `pytest` + `yaml` + `requests`

### 4. JSON/Groovy (CI/CD Pipeline Style) — БЕЗ Gherkin/Cucumber

**JSON Test Suite:**

```json
// python/tests/test_data/auth_suite.json
{
  "name": "Auth API Tests",
  "version": "1.0",
  "base_url": "http://localhost:8000/api",
  "tests": [
    {
      "id": "AUTH-001",
      "name": "Login valid",
      "method": "POST",
      "path": "/auth/login",
      "body": {
        "email": "alice@buzzhive.com",
        "password": "alice123"
      },
      "expected": {
        "status": 200,
        "field": "access_token"
      }
    },
    {
      "id": "AUTH-002",
      "name": "Login wrong password",
      "method": "POST",
      "path": "/auth/login",
      "body": {
        "email": "alice@buzzhive.com",
        "password": "wrongpassword"
      },
      "expected": {
        "status": 401
      }
    }
  ]
}
```

```python
import json
import pytest
import requests

BASE_URL = "http://localhost:8000/api"

with open("test_data/auth_suite.json") as f:
    suite = json.load(f)

@pytest.mark.parametrize("test", suite["tests"])
def test_suite(test):
    response = requests.request(
        test["method"],
        f"{BASE_URL}{test['path']}",
        json=test.get("body")
    )
    assert response.status_code == test["expected"]["status"]
    
    if test["expected"].get("field"):
        assert test["expected"]["field"] in response.json()
```

**Groovy (Jenkins Pipeline):**

```groovy
// jenkins/auth_tests.groovy
def testCases = [
    [email: "alice@buzzhive.com", password: "alice123", expected: 200],
    [email: "alice@buzzhive.com", password: "wrongpassword", expected: 401],
    [email: "", password: "alice123", expected: 422]
]

testCases.each { tc ->
    def response = httpPost(
        url: "http://localhost:8000/api/auth/login",
        body: tc
    )
    assert response.status == tc.expected : "Test failed: ${tc.email}"
}
```

**Инструменты:**
- JSON: `pytest` + `requests`
- Groovy: Jenkins Pipeline + HTTP request step

## Как перейти с Gherkin на другой формат

### Вариант 1: Python (рекомендуемый старт)

```bash
# 1. Установить
pip install pytest requests pytest-yaml

# 2. Создать тесты ( см. выше )
python/tests/test_auth.py

# 3. Запустить
pytest python/tests/test_auth.py -v
# или
npm run test:python
```

### Вариант 2: YAML (для больших suite)

```bash
# 1. Создать YAML файлы
python/tests/test_data/auth.yaml

# 2. Запустить
pytest python/tests/test_yaml_runner.py -v
```

### Вариант 3: JSON (для CI/CD)

```bash
# 1. Создать JSON
python/tests/test_data/auth_suite.json

# 2. Запустить
pytest python/tests/test_json_runner.py -v
```

### Вариант 4: Groovy (для Jenkins)

```groovy
// В Jenkinsfile
stage('API Tests') {
    steps {
        groovy {
            script {
                def tests = readJSON file: 'test_data/auth_suite.json'
                tests.tests.each { test ->
                    def response = httpRequest(
                        url: "${BASE_URL}${test.path}",
                        httpMode: test.method,
                        requestBody: test.body as String
                    )
                    assert response.status == test.expected.status
                }
            }
        }
    }
}
```
